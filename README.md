# face-blurrer

一括で画像に写っている人の顔をぼかすツール

## 前提

- Python
- uv

## 動かし方

```
IN_DIR=<input_dir> OUT_DIR=<output_dir> make
```

入力ディレクトリにある jpeg, png ファイルを出力ディレクトリに顔をぼかして保存します。
入力ディレクトリのディレクトリ構造は出力ディレクトリでも保持されます。

処理は 3 つのフェーズに分かれていて、順に実行されます。

1. `detect`: 全画像の顔検出 (無人・時間がかかる) → `WORK_DIR/detections.json`
2. `review`: 行灯の顔かもしれないものを y/n で確認 (対話) → `WORK_DIR/decisions.json`
3. `render`: 顔をぼかして書き出し (無人) → `OUT_DIR`

`make detect` / `make review` / `make render` で個別に実行することもできます。
`WORK_DIR` (デフォルト `work`) には中間結果の JSON が保存されます。

## レビュー

行灯の顔かもしれないものは画像が表示されるので、行灯の一部であれば y キーを、
そうでなければ n キーを押してください。行灯であるものにはぼかしは入れません。

q キーまたはウィンドウを閉じると中断できます。回答済みの分は保存されているので、
もう一度実行すると続きから再開します。

## 中断・やり直し

- どのフェーズも中断できます。再実行すると続きから再開します (detect は検出済み
  ファイルを、render は出力済みファイルをスキップします)。
- レビューで押し間違えたときは `WORK_DIR/decisions.json` の該当エントリの
  `"andon"` を直してから、`OUT_DIR` の該当ファイルを消して `make render` を
  再実行してください (`--force` を付けると全ファイルを再生成します)。
  モデルの再実行は不要です。
- 検出の設定 (`--imgsz`, `--conf`) を変える場合は `WORK_DIR` を消して
  detect からやり直してください。

## EXIF・画質

- 出力 JPEG には元画像の EXIF (撮影日時・カメラ情報・GPS など) をコピーします。
  ただし EXIF 内サムネイルはぼかし前の画像が残ってしまうため削除します。
  Orientation は 1 に正規化されます (ピクセル自体が回転済みのため)。
- JPEG は品質 97 で再エンコードされます (`--jpeg_quality` で調整可)。

## オプション

```
uv run main.py --help
```

- `--device`: 推論デバイス。デフォルトは auto (Apple Silicon なら GPU/MPS)
- `--imgsz`: 推論解像度。デフォルト 1280。大きいほど小さい顔を拾えるが遅い
- `--conf`: 検出の信頼度閾値。デフォルト 0.25。下げると拾い漏れが減るが誤検出が増える
- `--jpeg_quality`: JPEG 出力品質。デフォルト 97
- `--force`: render 時に出力済みファイルも再生成する
